[#2415](https://github.com/multiversx/mx-sdk-rs/pull/2415) "Proxies based on ABI crate only" targets a brand-new `feat/abi-crate` branch — the only PR against it so far — and proposes framework-agnostic ABI and proxy types usable across multiple framework versions simultaneously. ### `mx-chain-proxy-go` picks up a quiet resource-exhaustion hardening fix

[#518](https://github.com/multiversx/mx-chain-proxy-go/pull/518) "Extra checks body size" adds a configurable max-request-body-size limit (1MB default) to the proxy's HTTP server via a `http.MaxBytesReader` middleware, with explicit unit tests for both the rejection and pass-through paths. Reported the same way as any other engineering PR: plainly described, no exploit detail, no advisory reference — ordinary defensive hardening. This is the third such quiet body/URL-handling hardening fix across the watchlist in the past week, after `mx-api-service` #1617 (SSRF) and `mx-chain-es-indexer-go`'s v1.9.3-fix2 (JSON-escaping) — worth keeping an eye on as a pattern, not a coincidence.
