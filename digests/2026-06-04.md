## Themes

### 1. UB in BoxedBytes::from_concat — latent heap-corruption vector for WeeAlloc contracts

[mx-sdk-rs #2395](https://github.com/multiversx/mx-sdk-rs/pull/2395) fixes `BoxedBytes::from_concat` calling `alloc(size=0)` when all input slices are empty, violating Rust's allocator contract. Severity varies by allocator:

- **WeeAlloc (high):** silent free-list corruption — a subsequent allocation can return memory aliasing live contract state (balances, owner addresses, storage keys). An attacker supplying empty byte arguments to an endpoint that calls `from_concat` unconditionally can trigger this path.
- **AllocationForbidden / default (medium):** reliable DoS — the contract aborts with `"memory allocation forbidden"` instead of executing.
- **LeakingAllocator / StaticAllocator64K (low):** technically UB; no practical corruption observed due to no-op dealloc.

The fix is a one-line early return (`if result_len == 0 { return BoxedBytes::empty(); }`) plus five regression tests. Not yet merged — will likely land as a v0.67 blocker.
