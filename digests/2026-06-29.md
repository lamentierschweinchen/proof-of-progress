## Themes

### Coordinated v1.11.8 release (Jun 23)

The queue that sat at 8 open PRs at last digest finally cleared enough to release. mx-chain-go [v1.11.8](https://github.com/multiversx/mx-chain-go/releases/tag/v1.11.8) and mainnet-config [v1.11.8.0](https://github.com/multiversx/mx-chain-mainnet-config/releases/tag/v1.11.8.0) went out within 7 minutes of each other on Jun 23, with devnet-config following an hour later. The releases were preceded by Jun 23 merge wave: [#7887](https://github.com/multiversx/mx-chain-go/pull/7887) (resume-on-error refactor), [#7870](https://github.com/multiversx/mx-chain-go/pull/7870) (subround block time check), plus the rc/supernova ↔ master sync PRs (#7888, #7889). Pattern matches previous releases: PRs drain → Merge to master → Tag → mainnet-config update, all same day.

### "Andromeda" codename surfaces

[#7890](https://github.com/multiversx/mx-chain-go/pull/7890) "when transitioning to Andromeda, request missing proofs" (BeniaminDrasovean, merged Jun 29) is the first commit referencing a post-Supernova upgrade. The body describes it as an edge-case fix in metachain/shard-header finality attestation sync — specifically, when a finality-attesting header is locally available but its proof is missing at the transition boundary. The use of "Andromeda" as a named transition state in production branch code (not a `[WIP]` or feature branch) implies it is a planned hard-fork epoch upgrade, not a codename borrowed from a side project. Supernova is still in progress (the [#259](https://github.com/multiversx/mx-chain-mainnet-config/pull/259) v2.0.0.0 mainnet activation PR is still open at day 172); Andromeda comes after it.

## Strategic read

1. **v1.11.8 is the current mainnet target.** The Jun 23 coordinated release (chain-go + mainnet-config + devnet-config same day) follows the established deployment pattern precisely. The next ordinary release in this series would be v1.11.9; watch for PRs draining and a same-day tag.

2. **Andromeda changes the roadmap picture.** The implicit sequence was: Supernova activates via mainnet-config #259 → chain enters new era. The emergence of "Andromeda" as a named codebase-level transition (not a WIP) implies a third upgrade epoch is already in design. Given that Supernova itself took months from code-complete to mainnet activation, Andromeda planning now is not premature — it's the normal lead time.

3. **Supernova mainnet activation remains gated on mainnet-config #259 (day 172).** The v1.11.8 release did not advance this PR. It is the single bottleneck: chain-go code is ready, the `rc/supernova` branch is in sync, but the mainnet-config PR that would trigger the validator upgrade flag has not moved since Jan 8. When this merges, expect same-day v2.0.0.0 + v1.11.8.0 tag coordination.

4. **CVE-2025-64170 enters day 40.** The rpassword bump is orphaned — skipped in v0.66.1, v0.66.2, and absent from the v0.67 RC branch as far as can be determined from open PRs. If it doesn't ship in v0.67, a standalone v0.66.3 patch will be needed. Risk keeps accumulating with each release that bypasses it.

5. **v0.67 appears close.** Interactor config merged, workspace deps consolidated, naming cleanup PR open — these look like final housekeeping before tagging. The remaining open items ([#2404](https://github.com/multiversx/mx-sdk-rs/pull/2404) Docker/CI, [#2302](https://github.com/multiversx/mx-sdk-rs/pull/2302) ABI crate) may or may not block the tag. Watch #2378 for a merge date; a tag within 7–10 days is plausible.

6. **sasurobert day 26 silence; mx-chain-vm-go essentially frozen.** The VM core contributor remains publicly absent (last event: Jun 3 WatchEvent). With Supernova's VM work presumably done, this may be deliberate — but Andromeda VM changes, if any, will require VM-go updates. Worth monitoring when activity resumes.
