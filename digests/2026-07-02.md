## Themes

### mx-chain-go: BLS v2 consensus hardening in concentrated burst

Two PRs merged today both target the BLS v2 consensus path (ssd04 on time-remaining logic, sstanculeanu on dynamic subround boundaries), and three more are queued. The cluster — [#7892](https://github.com/multiversx/mx-chain-go/pull/7892), [#7893](https://github.com/multiversx/mx-chain-go/pull/7893), [#7894](https://github.com/multiversx/mx-chain-go/pull/7894), [#7895](https://github.com/multiversx/mx-chain-go/pull/7895), [#7896](https://github.com/multiversx/mx-chain-go/pull/7896) — together cover: atomic waitgroup safety in consensus state, round-aware signature tracking, dynamic subround boundary calculation, time-remaining validation, and extra diagnostic logs. This is a coordinated hardening of the consensus layer's BLS v2 path, not opportunistic cleanup. The pattern (5 PRs from 2 authors in 48h, all in `consensus/spos/bls/v2/`) is more disciplined than the broad burst during the v1.11.8 push last month.

### mx-sdk-rs: VM storage semantics tightened

[#2411](https://github.com/multiversx/mx-sdk-rs/pull/2411) is the most substantive new item on the v0.67 queue since the workspace-deps consolidation wave finished last week. The `AccountData::storage_get/storage_set` refactor enforces a single access path across the VM and scenario layers, and the `QueueMapper::clear()` regression test catches a real class of bugs (residual metadata after clear). Empty-write-as-delete semantics also match on-chain behavior more accurately. This is correctness infrastructure, not polish — worth watching for merge timing relative to v0.67 tag.

### mx-api-service: filter default-value bugfix

[#1612](https://github.com/multiversx/mx-api-service/pull/1612) (stefangutica) is a targeted fix: event and tx filter fields were being serialized with default values (empty string / zero) into Elasticsearch queries, causing false positives in WebSocket subscriptions. Fields are now `undefined` when not set by the caller. Small but the kind of bug that silently corrupts stream data for subscribers.

### org-sweep: mx-ai-skills and template-dapp variants active today

Outside the watchlist: `mx-ai-skills` received a commit today (Sorin Petreasca improving SC audit skills); `mx-template-dapp-solidjs` pushed at 13:16 UTC.
