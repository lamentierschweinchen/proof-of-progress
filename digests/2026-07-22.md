### `mx-chain-es-indexer-go` quietly patches a JSON-injection edge case, backported to an old line

[v1.9.3-fix2](https://github.com/multiversx/mx-chain-es-indexer-go/releases/tag/v1.9.3-fix2) shipped as a prerelease tagged off `v1.9.3-fix1` (Dec 2025) — not the current `v1.10.2` stable. The one real change, [commit 2e6fbde](https://github.com/multiversx/mx-chain-es-indexer-go/commit/2e6fbde), escapes NFT-update and transaction-hash identifiers before they go into the raw Elasticsearch bulk-update metadata JSON, closing off a path where an unescaped `_id` value could corrupt or hijack that metadata line. Framed the same way as any other engineering fix here: the diff is small, public, and self-explanatory, no advisory attached. What's notable is *where* it landed — an old `-fix` line rather than current stable, implying some deployment is still running 1.9.3.x and needed a targeted patch rather than an upgrade.
