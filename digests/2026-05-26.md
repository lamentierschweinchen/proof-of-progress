## Themes

### 1. v0.67 RC opens four days after v0.66.0

[#2378](https://github.com/multiversx/mx-sdk-rs/pull/2378) (andrei-marinica, opened May 25) — the v0.67 release candidate appeared one business day after the Bech32 refactor merged. The gap between v0.66.0 (May 21) and a new RC is unusually short even by this team's standards; it signals that the Bech32 API change was scoped for a minor bump during v0.66 development and the team is not waiting to accumulate more features. Watch for the CVE-2025-64170 rpassword fix to land here rather than as a standalone v0.66.1.

### 2. Bech32 HRP refactor — API-breaking address change

[#2377](https://github.com/multiversx/mx-sdk-rs/pull/2377) (andrei-marinica, merged May 25) — introduces `Bech32Hrp` as a first-class newtype, migrates `Bech32Address`, `Address::to_bech32`, network config, snippet interactors, wallet, and example code to use it, and extracts `Bech32AddressError` into its own module. Also bumps the Rust toolchain and CI workflow versions. Smart contract crates using `Address::to_bech32` or constructing Bech32 addresses directly will need updating. This is the kind of typing improvement that prevents silent network mismatches (e.g., using an erd1 address on a chain expecting a different HRP).

### 3. mx-chain-go fully stalled — Supernova blocker cluster intact

Zero PRs merged, zero commits to tracked branches in mx-chain-go since May 23. The full set of Supernova testnet blockers is unchanged: timestamp PRs (#7842, #7846), the outgoing-tx check (#7841), the feat/testnet-fixes → rc/supernova loop (#7853). The new PR [#7856](https://github.com/multiversx/mx-chain-go/pull/7856) (miiu96, May 25) adds extra epoch-change validation — a useful addition, but it goes into the queue rather than clearing it. The Supernova testnet CI pass cannot close until at least the timestamp cluster merges.

### 4. mx-exchange-service: third wave of dep upgrades in one week

[#1763](https://github.com/multiversx/mx-exchange-service/pull/1763) + [#1764](https://github.com/multiversx/mx-exchange-service/pull/1764) (claudiulataretu, May 25) — data-layer and app-level dependency upgrades following last week's Node 24 / TS 5.9 / ESLint 9 overhaul. ### 5. CVE-2025-64170 window widens to day 6

[#2374](https://github.com/multiversx/mx-sdk-rs/pull/2374) (dependabot) — the rpassword 7.3.1 → 7.5.0 bump fixing CVE-2025-64170 remains unmerged at day 6 with no v0.66.1 tag. ## Strategic read

1. **v0.67 will ship fast.** The Bech32 refactor is the core change; the CVE patch and toolchain bump travel with it. Given the v0.66 → v0.67 RC gap of four days, a v0.67 tag in the next 1–2 weeks is plausible — especially if no further breaking changes are queued.
2. **mx-chain-go stall is now noteworthy.** The repo was the most active watchlist member through late April and early May. Three straight days of zero merges while a 5-PR blocker cluster sits open suggests the team hit a harder-than-expected coordination problem in the Supernova testnet calibration. Watch #7853 (the integration PR) — its CI state is the canary.
3. **Address API is changing; dApp/SDK consumers should prepare.** `Bech32Hrp` is a correctness improvement, but any code that calls `Address::to_bech32` or constructs Bech32 strings from addresses will break at v0.67. SDK users who started on v0.66 have a short window to review call sites.
4. **mx-exchange-service release is overdue.** Four dep-upgrade PRs in one week, combined with the Node 24 + TS 5.9 overhaul from last week, looks like a deployment gate clearing before a tagged release. The exchange service underpins xExchange; a release here has downstream implications for the DEX frontend.
5. **@andrei-marinica's concentration (42% of 28d commits across all branches) is a single-point-of-failure risk for the Rust SDK.** The SDK's development velocity is almost entirely one person. A v0.67 release followed quickly by BLS Windows fixes and the ABI crate draft (#2302) shows sustained output — but the project should watch contributor bus-factor as the SDK reaches maturity.
6. **Supernova remains on track but the clock is ticking.** The testnet blocker cluster has been open for 2–3 weeks. If it doesn't clear before the end of May, the Supernova mainnet timeline slips into June. The GRPC driver (#7845) and new epoch-check PR (#7856) add work to the queue without reducing it.
