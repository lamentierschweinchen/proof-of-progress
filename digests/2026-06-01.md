## Themes

### 1. v0.66.1 patch release: CVE fix breaks out of the v0.67 gate

[#2389](https://github.com/multiversx/mx-sdk-rs/pull/2389) "Release: sc 0.66.1" opened at 21:37 UTC on May 31 — immediately after the three feature PRs landed. This is a clear strategy pivot: rather than holding CVE-2025-64170 ([#2374](https://github.com/multiversx/mx-sdk-rs/pull/2374), day 12) behind the rc/v0.67 multi-PR gate, @andrei-marinica is shipping a narrow patch that carries the wasm/Rust 1.96 fix (#2386), the sc-meta toolchain edge case (#2388), and almost certainly the rpassword CVE bump (#2374). v0.67 features (wallet reorg #2385, time types #2387, ABI crate #2302) may follow in a separate tag once the patch clears.

### 2. Rust 1.96 compatibility: wasm imports patched

[#2386](https://github.com/multiversx/mx-sdk-rs/pull/2386) addresses a breaking change in Rust 1.96: `extern "C"` wasm imports now require an explicit `#[link(wasm_import_module = "env")]` attribute. Without it, the wasm adapter silently binds to the wrong module. This is a compiler-version correctness fix — contracts built with Rust 1.96 without it would produce runtime import errors on-chain.

### 3. Time-native smart contracts: timestamp types in ManagedVec

[#2387](https://github.com/multiversx/mx-sdk-rs/pull/2387) implements `ManagedVecItem` for `TimestampMillis`, `TimestampSeconds`, `DurationMillis`, and `DurationSeconds` as fixed 8-byte payload items. These types can now be stored in `ManagedVec` and participate in derive-generated codec logic, enabling time-aware contract state without manual serialization boilerplate.

### 4. sc-meta toolchain stability: patch-version mismatch fixed

[#2388](https://github.com/multiversx/mx-sdk-rs/pull/2388) fixes an edge case in `sc-meta` where rustup toolchain arguments included the patch component (e.g., `1.85.1`), causing "Missing manifest" failures when only the `major.minor` form was registered. The `RustcVersion` struct is refactored to derive string representations on demand rather than caching a `short_string` field. Low headline value, high reliability value for CI pipelines.
