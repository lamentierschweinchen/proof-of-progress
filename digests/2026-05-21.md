## Themes

### 1. mx-sdk-rs v0.66.0 released

[#2373](https://github.com/multiversx/mx-sdk-rs/pull/2373) is the release merge PR, closed May 20 at 23:53 UTC. Tag `v0.66.0` is confirmed. The release bundles:
- **Reproducible builds** ([#2355](https://github.com/multiversx/mx-sdk-rs/pull/2355) + [#2369](https://github.com/multiversx/mx-sdk-rs/pull/2369)): 10 `sc-meta reproducible-build` sub-commands covering local-build, docker-build, source-pack, source-unpack, publish, check, download, etc. Path-traversal protection via `safe_join` in `source_unpack.rs`. Docker support via pinned build image + helper scripts.
- **Interactor UX** ([#2371](https://github.com/multiversx/mx-sdk-rs/pull/2371)): post-deploy and post-call output now prints MultiversX Explorer transaction + contract URLs when chain ID is recognized.
- **Masked wallet input** ([#2367](https://github.com/multiversx/mx-sdk-rs/pull/2367)): `sc-meta wallet new` now uses `rpassword` to mask keystore password entry.
- **sc-meta new `--overwrite`** ([#2368](https://github.com/multiversx/mx-sdk-rs/pull/2368)): template renaming and Cargo.toml rename logic fixed; `--overwrite` allows re-running new over an existing dir.
- Post-release patch already: [#2375](https://github.com/multiversx/mx-sdk-rs/pull/2375) adder.snippets.sh fix merged same night.

### 2. Security: CVE-2025-64170 in rpassword dependency

[#2374](https://github.com/multiversx/mx-sdk-rs/pull/2374) (dependabot, open) bumps `rpassword` from 7.3.1 → 7.5.0. rpassword 7.5.0 explicitly fixes **CVE-2025-64170**. The v0.66.0 release shipped with 7.3.1 (introduced in #2367). Impact is scoped to `sc-meta wallet new` users who create keystores interactively on the CLI. The fix is one dependabot PR away; expect a v0.66.1 or a manual accept-and-merge before the week is out.

## Strategic read

1. **v0.66.0 is the biggest SDK release in months.** Reproducible builds alone are a structural shift: on-chain contract verification is now first-class, closing a long-standing trust gap between deployed WASM bytecode and source code. The verifier service integration (`publish`, `check`, `download` sub-commands) means this isn't just a build tool — it's an attestation pipeline.
2. **The CVE-2025-64170 exposure is real but limited.** rpassword affects password masking on CLI wallet creation only. No on-chain or remote-execution attack surface. But a point release (v0.66.1) is likely; watch the repo within 48 hours.
3. **Delegation SC adopting v0.66 same day is a strong signal of readiness.** The fact that the core contributor (andrei-marinica) immediately upgraded the highest-profile system contract suggests v0.66 was well-validated before tagging. Other system contracts (`mx-bridge-eth-go`, governance SCs) should follow in days.
4. **`feat/testnet-fixes` → `rc/supernova` CI failure is the current gate.** Until #7853 merges, the three-layer release staging (testnet → rc → mainnet-config) has a gap. The stalled timestamp PRs (#7842, #7846) and the outgoing-tx inclusion check (#7841) are likely gating the loop.
5. **sasurobert's ZK crypto PRs remain dormant this window.** The open PRs in mx-chain-vm-go (#969 ZK crypto integration, #971 ZK review) haven't moved in days. ZK execution is a longer-horizon bet; no signal it's blocking anything near-term.
6. **`blockchain-school` is worth watching.** A new educational repo from MultiversX org could signal a formal developer-education initiative. Low signal for now, but if course materials start appearing it may affect developer onboarding to the v0.66 SDK.
