## Themes

### mx-chain-go execution-context cluster — 3 PRs in one day

- [#7857](https://github.com/multiversx/mx-chain-go/pull/7857) **White list miniblocks from execution results MetaHeaderV3** (miiu96) — `doWhitelistWithMetaBlockIfNeeded` now handles MetaHeaderV3 mini-blocks; cross-shard whitelist was previously blind to V3 execution results, which is a requirement for Supernova.
- [#7867](https://github.com/multiversx/mx-chain-go/pull/7867) **Update chain handler** (ssd04) — consolidates `SetLastExecutionInfo` to be called once atomically rather than from scattered call sites; paired with mx-chain-core-go [#438](https://github.com/multiversx/mx-chain-core-go/pull/438) (same author, same merge minute). Multi-repo synchronized drop confirms these are a single logical change.
- [#7866](https://github.com/multiversx/mx-chain-go/pull/7866) **Update context for replaced header optimization** (ssd04) — adds an early-return guard: if the execution result context is already set, skip redundant work. Together with #7867, this makes the execution context handling both atomic *and* idempotent — two key safety properties for state transitions at the Supernova epoch boundary.
- mx-chain-go 7-day commit velocity: **89 commits** (vs. 58 last week, 51 the week before). The 4-week trend of 40 → 19 → 51 → 89 is a clear acceleration into the final integration sprint.

### mx-sdk-rs v0.66.2 — stable-track patch while v0.67 waits

- [v0.66.2](https://github.com/multiversx/mx-sdk-rs/releases/tag/v0.66.2) tagged Jun 18 via [PR #2403](https://github.com/multiversx/mx-sdk-rs/pull/2403) — previous day's [#2402](https://github.com/multiversx/mx-sdk-rs/pull/2402) fix folded directly into a patch release rather than batched into v0.67. This is the pragmatic call: developers hitting the DebugApi bug in production get a fix today. The cost is maintaining two active tracks (0.66.x and 0.67-RC) simultaneously, but @andrei-marinica has done this before.
- `ManagedVec::clone_range` (renamed from `slice`, deprecated) + `requires_drop()` environment decoupling are API-clarity improvements that simplify correct usage in DebugApi contexts.

### mx-api-service — ffmpeg 8.0 compatibility

- [#1610](https://github.com/multiversx/mx-api-service/pull/1610) **fix video thumbnail generation for limited-range YUV videos** (claudiulataretu) — ffmpeg 8.0 introduced strict rejection of limited-range YUV input in its JPEG encoder. Video NFTs recorded on iOS were silently failing to generate thumbnails. Switching the intermediate frame output to PNG sidesteps the encoder restriction. Practical improvement for the NFT ecosystem.
