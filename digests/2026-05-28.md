## Themes

### 1. mx-chain-go v1.11.6 — mainnet validator upgrade ships

[v1.11.6](https://github.com/multiversx/mx-chain-go/releases/tag/v1.11.6) (published May 27, paired with config [v1.11.6.0](https://github.com/multiversx/mx-chain-mainnet-config/releases/tag/v1.11.6.0)) contains two PRs: "flags compressed" ([#7835](https://github.com/multiversx/mx-chain-go/pull/7835), sstanculeanu, merged Apr 23) and a linter fix ([#7859](https://github.com/multiversx/mx-chain-go/pull/7859), AdoAdoAdo, merged May 27). The substantive change is #7835: compressed block flag handling, which was flagged in earlier digests as a potential advisory signal given its generic title. The linter fix was the final CI gate that triggered the release. A paired mainnet-config tag means node operators must update configuration — this is not an optional binary-only upgrade. The 34-day gap from v1.11.5 (Apr 23) to v1.11.6 is longer than usual; the delay likely reflects wait for testnet validation.

### 2. ed25519-dalek replaces custom signing in rc/v0.67

[#2382](https://github.com/multiversx/mx-sdk-rs/pull/2382) (andrei-marinica, opened May 27 into rc/v0.67) replaces the custom ed25519 implementation with the `ed25519-dalek` crate — confirmed by the commit message in stats.json: *"signing: replaced custom ed25519 implementation with ed25519-dalek."* `ed25519-dalek` is widely audited and maintained by the Ristretto group, eliminating a bespoke cryptographic signing path. Combined with CVE-2025-64170 (rpassword bump, [#2374](https://github.com/multiversx/mx-sdk-rs/pull/2374), day 8 unmerged), v0.67 will ship two distinct cryptographic-library improvements. This PR is still open; its merge is the last known gate before the RC can be tagged.

### 3. Wallet refactor completes for rc/v0.67

[#2381](https://github.com/multiversx/mx-sdk-rs/pull/2381) (andrei-marinica, merged May 27 into rc/v0.67) — closes the wallet refactoring arc that began with the Bech32 HRP newtype ([#2377](https://github.com/multiversx/mx-sdk-rs/pull/2377)) and the wallet convert integration tests ([#2379](https://github.com/multiversx/mx-sdk-rs/pull/2379)). With wallet refactor merged and ed25519-dalek (#2382) open, the v0.67 RC is one PR away from having its full feature set.

### 4. mx-bls-rs v0.2.1 — Windows BLS unblocked

[#7](https://github.com/multiversx/mx-bls-rs/pull/7) "Windows build fix + CI" + [#8](https://github.com/multiversx/mx-bls-rs/pull/8) "Release: multiversx-bls v0.2.1" (both andrei-marinica, May 27). The Windows AMD64 static library was incorrectly linked; `stdc++` linkage is now conditional on the GNU toolchain. Also upgrades to Rust edition 2024. mx-bls-rs wraps `herumi/bls` for MultiversX's BLS cryptography — this fix is a prerequisite for cross-platform developer and CI tooling.

### 5. P2P and communication layer hardening

[mx-chain-communication-go #79](https://github.com/multiversx/mx-chain-communication-go/pull/79) (AdoAdoAdo, May 27) adds a deadline to direct P2P sends, preventing unbounded blocking on unresponsive peers. [mx-chain-core-go #435](https://github.com/multiversx/mx-chain-core-go/pull/435) (sstanculeanu, May 27) extends the `P2PPeerInfo` structure. Neither repo is on the primary watchlist but both feed into mx-chain-go: sstanculeanu's open PR [#7858](https://github.com/multiversx/mx-chain-go/pull/7858) will consume the updated communication-go dependency. This pattern — hardening P2P primitives before a major protocol upgrade — fits the Supernova preparation timeline.

## Strategic read

1. **Node operators: upgrade to v1.11.6 now.** The paired mainnet-config tag means this is a full validator upgrade — binary + config. The "flags compressed" protocol change requires the new configuration to take effect.
2. **v0.67 is one PR from feature-complete.** ed25519-dalek (#2382) is the last known gate on rc/v0.67. Once it merges, the RC has: Bech32 HRP refactor, wallet refactor, ed25519-dalek signing, and rpassword CVE fix. With the CVE at day 8, the tag should follow within days of #2382 merging.
3. **The Supernova blocker cluster is now in week 3.** v1.11.6 shipped to the mainnet (v1.11.x) track, not Supernova. The five-PR blocker cluster (#7841, #7842, #7846, #7853, #7857) and the GRPC driver [#7845](https://github.com/multiversx/mx-chain-go/pull/7845) remain unmerged. Supernova's June timeline is at risk.
4. **Two cryptographic library swaps converging on v0.67.** The shift from custom ed25519 to ed25519-dalek, combined with rpassword remediation, suggests the team is using the v0.67 release to modernize the SDK's cryptographic foundations. Downstream integrators using the wallet or signing APIs should test against rc/v0.67.
5. **P2P improvements suggest Supernova networking focus.** The direct-send deadline and P2PPeerInfo extension, both landing the same day as v1.11.6, suggest the mx-chain-go team is actively working on the networking layer — likely a parallel track to the Supernova consensus-layer blockers.
6. **@sasurobert remains inactive on tracked repos.** No contributions to mx-chain-vm-go or mx-chain-go in the window; the VM/consensus core work for Supernova is not publicly visible.
