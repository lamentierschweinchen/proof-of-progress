## Themes

### 1. mx-api-service v1.20.0 — 5-month release bundles Supernova readiness

[v1.20.0](https://github.com/multiversx/mx-api-service/releases/tag/v1.20.0) (published June 4 at 09:01 UTC by @stefangutica) is the first stable release since v1.19.0 on January 13 — a 143-day gap. The 23-PR changelog reveals two threads:

**Supernova integration:** [#1583](https://github.com/multiversx/mx-api-service/pull/1583) Supernova execution results mini-blocks, [#1586](https://github.com/multiversx/mx-api-service/pull/1586) new Supernova fields, [#1587](https://github.com/multiversx/mx-api-service/pull/1587) execution results blocks list, [#1590](https://github.com/multiversx/mx-api-service/pull/1590) dapp config chain-id override from network config — together these wire the API layer to consume and surface Supernova-specific block structure.

**Infrastructure hardening:** [#1606](https://github.com/multiversx/mx-api-service/pull/1606) DB connection reduction (54% peak connection drop, covered in yesterday's digest), [#1601](https://github.com/multiversx/mx-api-service/pull/1601) tx pool refresh every second with reduced TTL, [#1605](https://github.com/multiversx/mx-api-service/pull/1605) NestJS modules refactor eliminating circular dependency patterns, [#1600](https://github.com/multiversx/mx-api-service/pull/1600) package upgrades.

This is production Supernova-readiness work, not routine housekeeping. The 62-commit / 7-PR-merged 28-day window, driven entirely by @stefangutica, shows sustained acceleration since May.

### 2. mx-sdk-rs: UB and serde panics cleared in a single afternoon; v0.67 near

Two fixes merged June 4 sharpen the Rust SDK for v0.67:

- [#2395](https://github.com/multiversx/mx-sdk-rs/pull/2395) "Fix: UB in BoxedBytes::from_concat on Zero-Length Input" — the high-severity allocator UB (WeeAlloc heap-corruption path, identified yesterday) is now patched. One-line early return plus five regression tests. WeeAlloc contracts receiving empty-byte arguments are no longer silently vulnerable.
- [#2396](https://github.com/multiversx/mx-sdk-rs/pull/2396) "Bech32Address - serde deserialization error handling fix" — switches `Deserialize` impl from a potential-panic path to `try_from_bech32_string(...)` with proper serde error propagation. Adds full test coverage for legacy `bech32:` prefix and invalid-input cases.

With both #2395 and #2396 merged, @andrei-marinica has cleared the known correctness blockers. Only [#2394](https://github.com/multiversx/mx-sdk-rs/pull/2394) (interactor configs) stands between current master and a v0.67 tag. CVE-2025-64170 (#2374) remains the wild card at 16 days open.

## Strategic read

1. **mx-api-service v1.20.0 is a Supernova readiness milestone, not routine maintenance.** A 143-day gap followed by 23 PRs including explicit Supernova execution-results parsing signals that the API layer is now staged for Supernova mainnet traffic. Coordinate this release with the DeFi SDK releases and it reads as deliberate deployment sequencing.
2. **v0.67 is functionally ready; the delay is now editorial or policy.** Both correctness blockers (#2395 UB, #2396 serde) cleared in one afternoon. Only #2394 (interactor configs) and CVE-2025-64170 (#2374, day 16) remain. If #2374 is excluded again (as it was from v0.66.1), the tag could drop any day. A second CVE exclusion would establish a pattern worth calling out.
3. **CVE-2025-64170 is now at 16 days — the longest-running unresolved advisory in recent history.** If v0.67 tags without it, expect a v0.67.1 micro-patch, making this the third consecutive release cycle with a staggered security patch.
4. **The coordinated DeFi SDK release day is a leading indicator.** mx-sdk-dapp-liquidity, mx-sdk-dapp-swap, and mx-api-service all shipping on June 4 is not accidental. Watch mx-exchange-service and mx-exchange-sc for corresponding bumps in the next 1–3 days.
5. **The Supernova blocker cluster grew back to 10 with #7869.** @BeniaminDrasovean's block-proposal validation PR opens the same day #7868 merged. The cluster is not converging — it is being maintained at depth as the testnet-fixes branch accumulates correctness work.
6. **mx-chain-vm-go's silence (36 days, @sasurobert dark) now coincides with api-service gaining Supernova execution-results parsing.** If the VM layer is not keeping pace in public, either (a) Supernova's VM delta is smaller than advertised, (b) it is in a private branch, or (c) the VM-layer release is intentionally decoupled and will come as a tagged dependency bump rather than visible PRs. The api-service changes land before the VM changes are visible — that sequencing is odd.
